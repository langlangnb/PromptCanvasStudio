# PromptCanvas Studio

`PromptCanvas Studio` 是一个面向 `gpt-image-2` 及兼容网关的对话式生图 Web 工具。

它不是聊天机器人外壳，而是偏生产场景的图片工作台：支持对话线程、参考图粘贴、多图生成、电商多图优化、图片反推提示词、批量下载，以及本地浏览器缓存保存已生成图片。

## 特性

- 对话式生图
  - 通过线程方式组织提示词、参考图、生成结果和失败重试
- 支持参考图
  - 可直接在输入框中粘贴图片作为参考图
  - 支持 `images/generations` 和 `images/edits` 两类请求路径的兼容处理
- 支持多分辨率与多画幅
  - 分辨率：`1K / 2K / 4K`
  - 画幅：`1:1 / 16:9 / 9:16 / 3:4 / 4:3`
  - 请求时会直接把对应像素尺寸发送给模型
- 支持多图生成
  - 手动指定数量
  - 也可从提示词中解析数量，例如 `10张`、`1张主图+9张副图`
  - 支持 `排队生成` 和 `并行生成`
- 电商多图模式
  - 适合主图 + 副图场景
  - 可调用辅助文本模型先优化多图提示词，降低多张结果重复的问题
- 图片反推提示词
  - 支持粘贴图片后调用文本模型分析
  - 输出中文主提示词、英文主提示词、负向提示词、风格标签、画面拆解和变体建议
  - 反推结果可一键填回输入框继续编辑
- 失败重试
  - 单图失败可重试
  - 反推失败可重试
  - 重试时会复用上下文信息
- 结果管理
  - 点击图片放大预览
  - 支持单张下载
  - 支持多图结果一键批量下载
- 浏览器缓存保存
  - 已生成图片会缓存在浏览器 IndexedDB 中
  - 不落地到你的本地文件夹，除非你主动点击下载

## 当前支持的生成尺寸

程序会把分辨率和画幅映射为实际请求像素，并直接发送给模型。

### 1K

- `1:1` -> `1024x1024`
- `16:9` -> `1024x576`
- `9:16` -> `576x1024`
- `3:4` -> `768x1024`
- `4:3` -> `1024x768`

### 2K

- `1:1` -> `2048x2048`
- `16:9` -> `2048x1152`
- `9:16` -> `1152x2048`
- `3:4` -> `1536x2048`
- `4:3` -> `2048x1536`

### 4K

- `1:1` -> `4096x4096`
- `16:9` -> `4096x2304`
- `9:16` -> `2304x4096`
- `3:4` -> `3072x4096`
- `4:3` -> `4096x3072`

## 技术栈

- 前端：原生 `HTML + CSS + JavaScript`
- 后端：原生 `Node.js http server`
- 存储：
  - 设置和线程：浏览器 `localStorage`
  - 生成图片缓存：浏览器 `IndexedDB`

没有引入前端框架，也没有依赖额外 npm 包，部署和二次开发都比较直接。

## 设置说明

### API Endpoint / Base URL

可填：

- `https://api.openai.com/v1`
- 或兼容网关的基础地址
- 也可以直接填完整接口地址，程序会自动规范化到对应路径

程序会自动处理：

- 生图：`/images/generations`
- 带参考图编辑：`/images/edits`
- 辅助文本模型与反推：`/chat/completions`

### API Key

填你自己的密钥。

### 图片模型

默认：

```text
gpt-image-2
```

如果你使用的是兼容网关，且模型名不同，可以在设置里直接改。

### 辅助文本模型

默认：

```text
gpt-5.4-mini
```

用于：

- 电商多图模式下的提示词优化
- 图片反推提示词

### 辅助模型推理强度

可选：

- `low`
- `medium`
- `high`

默认是 `medium`。

## 接口说明

服务端当前提供以下接口：

- `GET /api/health`
  - 健康检查
- `POST /api/generate`
  - 生图请求
- `POST /api/assist-prompt`
  - 电商多图提示词优化
- `POST /api/reverse-prompt`
  - 图片反推提示词

静态资源由 `server.js` 直接托管。

## 使用流程

### 普通生图

1. 打开设置，填写 `API Endpoint`、`API Key`、模型名
2. 选择分辨率、画幅、数量
3. 输入提示词
4. 回车或点击“生成图片”

### 参考图生图

1. 直接在输入框中粘贴图片
2. 再补充文字要求
3. 发送生成

如果当前请求带参考图，服务端会优先走编辑类接口。

### 电商多图

1. 勾选“电商生图模式”
2. 填数量，或者在提示词中写 `1张主图+9张副图`
3. 选择：
   - `排队生成`
   - `并行生成`
4. 发送

说明：

- 排队生成支持暂停和恢复
- 并行生成不支持暂停和恢复
- 开启电商模式后，辅助文本模型会先生成差异化子提示词，再进入正式生图

### 图片反推提示词

1. 切换到“反推提示词”
2. 粘贴 1 张或多张图片
3. 可额外补充文字要求
4. 发送

反推结果出来后：

- 可复制提示词
- 可填回输入框
- 可继续手动修改后再生成

当前实现中，“继续生图”按钮只回填输入框，不会自动发起生图请求。

## 数据存储说明

这个项目默认不会把生成结果直接保存到你的本地目录。

会存储的数据：

- 设置：`localStorage`
- 对话线程：`localStorage`
- 已生成图片缓存：`IndexedDB`

这意味着：

- 刷新页面后，大部分历史仍可恢复
- 清浏览器数据后，缓存图片和线程也会一起消失
- 只有你点击下载时，文件才会保存到本地

## 已知说明

### 1. 兼容网关差异

不同兼容网关对以下接口的支持不完全一致：

- `/images/generations`
- `/images/edits`
- `/chat/completions`

有些网关可能：

- 支持纯文本生图，但不支持参考图编辑
- 支持 OpenAI 风格路径，但参数格式有差异
- 在上游超时或故障时返回 `502 / 504`

这不是前端页面本身一定有问题，很多时候是上游兼容性或可用性问题。

### 2. 大图与多图性能

`2K / 4K`、多图并行、参考图反推都会显著增加：

- 请求体积
- 浏览器内存占用
- 上游响应时间

如果你的网关稳定性一般，建议优先：

- 降低并发
- 改用排队生成
- 缩减参考图数量

### 3. 浏览器缓存不是永久存储

图片只是缓存在浏览器本地数据库中，不等于长期备份。

如果结果重要，请主动下载保存。
